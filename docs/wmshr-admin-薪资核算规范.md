# WMSHR Admin 薪资核算模块规范

## 1. 开发顺序

薪资核算模块按 v2 界面优先迁移，不允许从旧正式后台字段反推界面。

固定顺序：

1. 先迁移 `admin-v2` 的薪资核算可见界面。
2. 再根据界面字段重写/整理列表、详情、生成、重算、确认、发放等接口契约。
3. 最后对接接口并验证真实加载、生成、回显和操作状态。

## 2. 当前界面方向

薪资核算入口恢复后，应保持 v2 的业务视角：

- 月份工资发放与核算。
- 月度汇总卡片。
- 员工薪资结果列表。
- 发放/核对/确认等业务动作。
- 工资单或薪酬详情查看。

旧后台接口中存在的内部状态、兼容字段、历史规则字段，只能服务接口和计算，不得反向污染 v2 UI。

## 3. 弹窗结构

薪资核算模块所有详情、管理、编辑类弹窗都必须使用三段式结构：

```text
Header
Body
Footer
```

要求：

- Header 固定在顶部。
- Body 是唯一可滚动区域。
- Footer 固定在底部。
- 关闭、取消、保存、新增等主要按钮必须放在 Footer。
- 不允许把取消/保存按钮放在 Body 末尾，避免内容过长时滚动消失。

## 4. 接口对接原则

- 列表接口服务薪资核算主列表。
- 详情接口服务工资单/薪酬详情弹窗。
- 一次性薪资项接口服务薪资项管理弹窗。
- 生成、重算、核对、确认接口按业务按钮触发。
- 保存或操作完成后必须刷新当前月份结果，确认列表金额和状态回显。

## 5. 当前恢复状态

- 侧边栏已恢复 `薪资核算` 入口。
- `App.tsx` 已挂载 `PayrollTable`。
- 主界面已调整为 v2 方向：月份核算栏、快捷月份/自定义月份、4 个概览卡、待发放/已发放筛选、搜索、生成本月薪资、重算本月、导出 CSV。
- 薪资结果主体已按 v2 表格形态展示：员工详情、上班天数、有效工时、加班次数、总加班时长、计算基薪、加班费、税后实发、发放状态/动作。
- 不再使用卡片式薪资结果列表作为主界面；卡片结构不符合 v2 薪资核算页面。
- 薪资模块弹窗封装已支持固定 Footer。
- 薪酬详情、一次性薪资项管理、新增/编辑薪资项弹窗已按 Header / Body / Footer 调整。

## 6. 验证要求

每次改薪资核算模块后至少执行：

```bash
npm run lint
npm run build
```

如果涉及后端接口，还要执行：

```bash
node -e "import('./server/index.js').then(()=>console.log('server import ok'))"
```

如果服务在运行，还要确认：

```text
前端 200
受保护 API 未登录 401 属于正常
```
