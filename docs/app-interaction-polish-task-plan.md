# App 交互完善第一批任务安排

## 已确认范围
- 仅针对 app 当前已有功能的界面交互进行完善。
- 不新增业务模块。
- 第一批优先处理：
  1. 首页打卡闭环
  2. 考勤列表页闭环（不保留详情页）

## 任务拆解
1. 首页打卡闭环完善
   - 目标：让用户在首页能明确知道当前状态、下一步动作、打卡过程进度与失败后的恢复路径。
2. 考勤列表页闭环完善
   - 目标：取消详情页依赖，让考勤列表页直接承载查看、判断和状态表达。
3. 真实验证
   - 目标：通过 mobile 端实际 lint / 类型检查，确认交互改动不破坏现有工程。

## 依赖顺序
1. 先检查首页打卡闭环现状并落实改动
2. 再检查考勤列表页现状并落实改动
3. 最后统一验证

## 验收标准
- 首页打卡状态、异常提示、进行中状态更清晰。
- 考勤列表页不再依赖详情页完成查看。
- 工程通过 mobile 端真实校验。

## AFK / HITL
- 当前实施：AFK
- 如发现会实质改变业务口径的交互选择，再转 HITL 确认
