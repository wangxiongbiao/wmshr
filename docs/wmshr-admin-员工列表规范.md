# WMshr Admin 员工列表规范

## 1. 目标

员工列表页用于在 admin 后台中快速完成以下任务：

- 查看员工核心档案信息
- 搜索员工
- 进入员工编辑
- 通过 v2 的删除入口让员工退出当前列表

当前页面以 `admin-v2/src/components/EmployeeList.tsx` 的轻量卡片式员工档案列表为基准，不使用表格替代，也不保留正式后台增强版的多筛选、分页、停用/离职双按钮等信息架构。

---

## 2. 页面结构

员工列表页由 3 个区域组成：

1. 顶部搜索与新增区
2. 中部员工卡片列表区
3. 底部员工总数

页面整体保持现有 admin 主题风格：

- 使用 `glass-panel` 作为主容器基底
- 卡片圆角、边框、浅阴影延续当前 admin 视觉
- 操作按钮使用品牌蓝 `brand`
- 状态标签延续 v2 卡片标签风格

---

## 3. 顶部搜索区规范

### 3.1 布局

顶部严格按 v2 原型保持左右结构：

- 左侧：单个搜索框
- 右侧：`新增员工` 主按钮
- 不展示员工状态、国家/地区、计薪方式、岗位、考勤规则等正式后台多筛选项
- 不展示 `包含停用/离职` 复选项
- 不展示 `重置` 按钮

### 3.2 搜索范围

搜索关键词只匹配 v2 原型字段：

- 姓名
- 岗位
- 部门/区域

### 3.3 交互规则

- 搜索在当前已加载员工数据中即时过滤
- `新增员工` 为主按钮，始终保持明显视觉优先级

---

## 4. 员工卡片规范

### 4.1 卡片布局

必须保留 v2 卡片式布局，不改为大横条信息墙，也不改成表格。

每张卡片包含：

- 头像或姓名首字占位
- 员工姓名
- 国家/地区
- 性别
- 员工状态
- 职位
- 区域
- 时薪
- 基础工资（月薪）
- 币种
- 入职

不得把正式后台增强字段重新带回卡片主界面，例如：员工编号、手机号、当前考勤规则、计薪方式、薪资配置等。

### 4.2 操作按钮

卡片右上角 hover 显示操作按钮：

- 编辑
- 删除

删除入口沿用 v2 图标和文案。当前后端没有物理删除接口时，前端可通过状态接口让员工退出当前列表，但不得在卡片上恢复正式后台的停用/离职双按钮。

### 4.3 状态标签

员工状态按 v2 轻量标签显示：

- 在职：绿色倾向
- 休假/其他状态：蓝色倾向

---

## 5. 员工总数规范

底部只显示 v2 原型的员工总数：`共 N 名员工`。

员工管理界面不展示分页条、页码按钮、当前显示区间等正式后台增强版控件。

---

## 6. 加载与空状态规范

### 6.1 列表加载

员工列表首次加载时，可显示列表级 loading。

但以下场景不能使用列表 loading 替换整页：

- 点击编辑员工
- 打开员工详情类弹窗

这类场景应使用全局遮罩 loading。

### 6.2 空状态

当当前搜索条件下没有员工时，显示：

- 明确的空状态文案
- 不破坏顶部搜索区
- 不隐藏新增员工入口

建议文案：

`当前筛选条件下没有员工数据`

---

## 7. 编辑弹窗联动规范

### 7.1 打开编辑

点击卡片“编辑”后：

- 页面显示全局遮罩 loading
- 异步获取员工详情
- 详情返回后再打开弹窗

不能让员工列表本身进入整块 loading 态。

### 7.2 保存后回刷

员工新增、编辑、删除状态变更后：

- 更新当前已加载员工数组
- 保持 v2 卡片界面结构

---

## 8. 员工编辑弹窗规范

员工新增/编辑弹窗同样以 `admin-v2/src/components/Modals.tsx` 的员工弹窗为基准。

可见字段只包含：

- 照片
- 姓名
- 性别
- 来源国家
- 职位
- 所属区域
- 时薪
- 基础工资（月薪）
- 全勤奖
- 社保金
- 工资币种
- 入职日期

不得把正式后台增强字段重新显示到 v2 弹窗里，例如：员工编号、手机号、员工状态、计薪方式单选、薪资生效日期、薪资摘要侧栏、规则历史等。

现有后端仍需要的隐藏字段由前端提交默认值或保留旧值：手机号、员工状态、计薪方式、薪资生效日期等。隐藏字段只服务接口兼容，不得反向影响 v2 界面。

---

## 9. 后端接口规范

员工管理界面直接消费已加载员工数组；前端列表组件不再调用正式后台分页接口，也不依赖 `roleOptions`、`hasMore` 等分页筛选返回字段。

后端如仍保留分页接口，只作为兼容接口，不应反向影响 v2 员工管理界面。

---

## 10. 禁止事项

员工列表后续优化时，以下事项默认禁止：

- 不得把员工卡片改成横向信息墙
- 不得把员工列表默认改成表格
- 不得把正式后台多筛选区、分页条、员工编号、手机号、计薪方式、薪资配置、停用/离职双按钮重新带回 v2 员工管理界面
- 不得在点击编辑时让整个员工列表进入 loading 覆盖态

---

## 11. 当前基准说明

本文件以 `admin-v2/src/components/EmployeeList.tsx` 为员工管理列表界面的当前基准，作为后续设计对齐、前端改版、代码 review 和模块风格统一的参考标准。
