# 门户 / Admin / 登录页 国际化长文案兼容任务安排

## 已确认范围
1. 只覆盖 `apps/home` 门户、`apps/admin` 后台及其登录页。
2. 优先处理高风险展示位中的长文案兼容问题。
3. 不改业务接口和文案语义，只修正展示承载方式。
4. 完成后执行真实 lint / build 校验。

## 依赖顺序
1. 先定位门户、Admin、登录页的高风险组件与当前单行裁切点。
2. 再按组件类型做最小必要样式修正与注释补充。
3. 最后执行 lint / build，并把结果回填进度文档。

## 任务拆解
### Task 1：定位高风险组件
- 目标：确认门户、Admin、登录页里当前最容易被长文案打坏的真实组件与文件。
- 输出：目标文件清单、风险点清单。
- 验收标准：每个模块至少锁定 1 个核心高风险区域。
- 依赖：无
- 属性：AFK

### Task 2：修正门户长文案承载
- 目标：修正门户关键标题/描述/CTA 的长文案承载方式。
- 输入：Task 1 的门户风险点。
- 输出：门户相关样式或组件改动。
- 验收标准：核心文案可换行或自适应，不再依赖强制裁切。
- 依赖：Task 1
- 属性：AFK

### Task 3：修正 Admin 长文案承载
- 目标：修正 Admin 导航、标题、按钮或说明文案的长文案承载方式。
- 输入：Task 1 的 Admin 风险点。
- 输出：Admin 相关样式或组件改动。
- 验收标准：核心操作入口与标题在长文案下仍可读且不挤坏布局。
- 依赖：Task 1
- 属性：AFK

### Task 4：修正登录页长文案承载
- 目标：修正登录页主标题、说明和主按钮的长文案承载方式。
- 输入：Task 1 的登录页风险点。
- 输出：登录页相关样式或组件改动。
- 验收标准：登录页主操作区在长文案下仍稳定、可读、可点击。
- 依赖：Task 1
- 属性：AFK

### Task 5：验证与回填
- 目标：通过真实校验确认改动没有破坏工程，并回填执行结果。
- 输入：Task 2-4 的代码改动。
- 输出：验证结果、进度文档更新。
- 验收标准：`npm run lint`、`npm run build` 通过；进度文档已同步。
- 依赖：Task 2、Task 3、Task 4
- 属性：AFK
