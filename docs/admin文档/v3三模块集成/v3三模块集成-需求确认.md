# v3 三模块集成需求确认

## 1. 目标
将 `admin-v3` 中的 `customers`、`goods`、`expenses` 三个模块集成到当前 `apps/admin`，并以 **直接使用 v3 UI 原型** 为前提，只做系统通用适配与数据层接入。

## 2. 已确认范围
### 纳入本轮
- 客户管理（customers）
- 入库管理（goods）
- 费用管理 / 费用核销（expenses）

### 不纳入本轮
- 产品管理（products）
- 订单管理（orders）

## 3. 实施口径
1. 保留 v3 三模块页面主结构、列表布局、表单交互和视觉表达
2. 接入当前 admin 的：
   - `TabId`
   - `adminRoute`
   - `Sidebar`
   - `App.tsx`
   - `lib/api.ts`
   - `server/index.js`
3. 去除 `localStorage` 与 mock 数据作为真实数据源
4. 替换为真实 API + 服务端持久化
5. 只做系统通用适配，不做整页重设计

## 4. 验收标准
- 三模块可以从当前 admin 正式访问
- 页面主视觉仍然是 v3 风格
- 关键操作走真实 API
- 刷新后数据不丢失
- 不破坏现有 `dashboard / employees / attendance / payroll / sop`

## 5. 待后续确认但不阻塞 Phase 1 / 2 的业务问题
- 费用“核销”是否等于审批通过
- 附件/凭证首期是否必须正式上传
- 客户信用流水首期是否只读
