## 6. 当前阻塞 / 待补验收项

以下问题不阻塞当前代码实现落地，但会影响“最终业务验收”的完整度：

1. **当前浏览器自动化会话没有 admin 登录态**
   - 当前可验证到登录门禁、路由入口和匿名前端控制台
   - 还不能直接在浏览器里完成 `customers / goods / expenses` 的已登录新增/编辑/保存/刷新验收

2. **本机 `npx supabase` 在 Darwin x64 环境缺少可用 CLI 二进制**
   - 执行 `npx supabase migration list` 时失败：`No matching Supabase CLI binary package found for darwin-x64`
   - 因此当前无法直接通过本机 Supabase CLI 核对远端 migration 执行状态

当前默认推进口径：
- 附件字段首期继续使用 URL / data URL / 文本字段过渡
- 先确认源码、接口、构建与本地服务链路闭环
- 后续获得登录态后再补“页面内真实写操作”验收
