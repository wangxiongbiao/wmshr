# 薪酬计算管理模块后端开发设计文档

## 1. 文档目标

本文档用于指导 `wmshr-admin` 薪酬计算管理模块的后端数据模型、接口、计算服务和业务约束设计。

设计原则：

- 后端负责统一薪酬计算口径，前端只展示后端结果。
- 薪酬计算以自然月为周期。
- 薪酬计算读取 `MonthlyAttendanceSummary`，不直接读取原始考勤记录。
- 固定工资和时薪二选一。
- 一次性薪资项只作用于当前自然月。
- 本模块不维护员工基础档案、不配置考勤规则、不计算每日考勤。

关联文档：

- `admin文档/薪酬计算管理模块/薪酬计算管理模块-字段与功能梳理.md`
- `admin文档/薪酬计算管理模块/薪酬计算管理模块-前端开发设计文档.md`
- `admin文档/考勤计算管理模块/考勤计算管理模块-后端开发设计文档.md`

## 2. 已确认业务规则

### 2.1 薪酬周期

- 薪酬周期为自然月。
- 所有薪酬结果按员工和年月生成。

### 2.2 薪酬计算输入

薪酬计算读取：

- 员工薪资基础配置。
- 自然月考勤汇总。
- 当前自然月一次性薪资项。

不直接读取：

- 原始打卡记录。
- 每日临时前端计算结果。

### 2.3 计薪方式互斥

- 固定工资和时薪只能二选一。
- `salaryType = fixed` 时只使用 `fixedSalary`。
- `salaryType = hourly` 时只使用 `hourlyRate`。

### 2.4 税率处理

- 当前业务文档未确认税率规则。
- 本模块不把 `taxRate` 作为核心薪酬字段。
- 如需税费，先作为扣款项录入 `SalaryAdjustmentItem`。

### 2.5 阻断规则

如果薪酬计算缺少关键输入，应生成阻断结果，而不是静默跳过。

阻断结果：

- `calculationStatus = blocked`
- 写入 `blockedReason`
- 不允许确认薪酬

## 3. 数据模型设计

## 3.1 SalaryProfile 员工薪资基础配置表

### 表定位

保存员工薪资基础配置，包括计薪方式、固定工资、时薪和币种。

### 字段定义

#### `id`

- 名称：薪资配置 ID
- 类型：`number` / `bigint` / `uuid`
- 是否必填：是
- 生成方式：后端生成

#### `employeeId`

- 名称：员工 ID
- 类型：`number` / `bigint` / `uuid`
- 是否必填：是
- 关联：`Employee.id`

#### `salaryType`

- 名称：计薪方式
- 类型：`enum`
- 枚举值：`fixed | hourly`
- 是否必填：是

#### `fixedSalary`

- 名称：固定工资金额
- 类型：`decimal | null`
- 规则：`salaryType = fixed` 时必填，`salaryType = hourly` 时为空或不参与计算

#### `hourlyRate`

- 名称：时薪金额
- 类型：`decimal | null`
- 规则：`salaryType = hourly` 时必填，`salaryType = fixed` 时为空或不参与计算

#### `currency`

- 名称：币种
- 类型：`enum`
- 枚举值：`THB | USD | MYR | IDR`
- 是否必填：是

#### `isActive`

- 名称：是否启用
- 类型：`boolean`
- 默认值：`true`

#### `effectiveStartDate`

- 名称：生效开始日期
- 类型：`date | null`
- 说明：后续支持薪资配置历史时使用

#### `effectiveEndDate`

- 名称：生效结束日期
- 类型：`date | null`
- 说明：为空表示长期有效

#### `createdAt` / `updatedAt`

- 后端维护

### 约束

- 同一员工只能有一个当前启用薪资配置。
- 固定工资和时薪必须互斥。
- 金额不能小于 0。

---

## 3.2 SalaryAdjustmentItem 一次性薪资项表

### 表定位

保存员工某自然月的一次性薪资项，包括补贴、扣款、其他。

### 字段定义

```ts
interface SalaryAdjustmentItem {
  id: string | number;
  employeeId: string | number;
  yearMonth: string;
  type: 'allowance' | 'deduction' | 'other';
  name: string;
  amount: number;
  note: string;
  createdAt: string;
  updatedAt: string;
}
```

### 约束

- `employeeId` 必须存在。
- `yearMonth` 必须为 `YYYY-MM`。
- `type` 必须为合法枚举。
- `name` 去除首尾空格后不能为空。
- `amount` 必须大于等于 0。
- 一次性薪资项只作用于所属员工和所属自然月。

---

## 3.3 MonthlyPayrollResult 自然月薪酬结果表

### 表定位

保存员工某自然月薪酬计算结果，是薪酬计算管理模块的核心结果表。

### 字段定义

```ts
interface MonthlyPayrollResult {
  id: string | number;
  employeeId: string | number;
  yearMonth: string;

  salaryType: 'fixed' | 'hourly';
  fixedSalary: number | null;
  hourlyRate: number | null;
  currency: 'THB' | 'USD' | 'MYR' | 'IDR';

  validHours: number;
  standardHours: number;
  hourlyPay: number;
  overtimePayHours: number;
  overtimePay: number;

  allowanceTotal: number;
  deductionTotal: number;
  otherTotal: number;
  grossPay: number;
  totalDeduction: number;
  netPay: number;

  calculationStatus: 'draft' | 'calculated' | 'blocked' | 'confirmed';
  reviewStatus: 'pending' | 'approved' | 'rejected';
  blockedReason: string | null;
  calculatedAt: string | null;
  confirmedAt: string | null;
  createdAt: string;
  updatedAt: string;
}
```

### 唯一约束

建议：

```text
employeeId + yearMonth 唯一
```

同一员工同一自然月只保留一条当前薪酬结果。重新计算时更新该结果。

## 4. 薪酬计算服务设计

## 4.1 单员工月度薪酬计算输入

```ts
interface CalculateMonthlyPayrollInput {
  employeeId: string | number;
  yearMonth: string;
}
```

## 4.2 单员工月度薪酬计算流程

1. 查询员工。
2. 查询员工当前或当月有效薪资配置 `SalaryProfile`。
3. 校验固定工资和时薪互斥。
4. 查询员工该月 `MonthlyAttendanceSummary`。
5. 如果考勤汇总不存在或不可进入薪资核算，生成阻断结果。
6. 查询员工该月 `SalaryAdjustmentItem` 列表。
7. 按类型汇总补贴、扣款、其他。
8. 根据计薪方式计算固定工资或时薪工资。
9. 计算加班费。
10. 计算应发金额。
11. 计算扣款金额。
12. 计算实发金额。
13. 保存或更新 `MonthlyPayrollResult`。
14. 返回薪酬结果。

## 4.3 固定工资计算

```text
basePay = fixedSalary
hourlyPay = 0
```

说明：固定工资员工本轮仍允许叠加加班费。

## 4.4 时薪工资计算

建议采用不重复计薪口径：

```text
standardPaidHours = max(0, validHours - overtimePayHours)
hourlyPay = standardPaidHours × hourlyRate
```

说明：

- 当前代码实际方向是“标准内有效工时工资 + 加班费”。
- 如业务后续确认按全部有效工时计薪，应单独调整此公式。

## 4.5 加班费计算

```text
overtimePay = overtimePayHours × overtimeHourlyFee
```

注意：

- `overtimePayHours` 来自 `MonthlyAttendanceSummary.totalOvertimePayHours`。
- 加班费标准建议由考勤计算结果或月度考勤汇总固化后提供。
- 如果暂未固化，可临时从当月规则计算结果汇总，但不建议薪酬模块重新匹配考勤规则。

## 4.6 一次性薪资项汇总

```text
allowanceTotal = sum(type = allowance)
deductionTotal = sum(type = deduction)
otherTotal = sum(type = other)
```

`other` 本轮默认按正向加项处理。

## 4.7 应发、扣款、实发计算

```text
grossPay = basePay 或 hourlyPay + overtimePay + allowanceTotal + otherTotal
totalDeduction = deductionTotal
netPay = grossPay - totalDeduction
```

### 4.8 阻断结果生成

阻断场景：

- 员工不存在。
- 薪资配置缺失。
- 固定工资和时薪配置冲突。
- 固定工资和时薪金额均为空。
- 月度考勤汇总缺失。
- 月度考勤汇总不可进入薪资核算。
- 币种缺失。

阻断结果字段：

```text
calculationStatus = blocked
reviewStatus = pending
blockedReason = 具体原因
金额字段默认为 0 或保留可追溯输入
```

## 5. 接口设计

## 5.1 获取薪酬结果列表

### 请求

```http
GET /api/admin/payroll-results
```

### 查询参数

- `yearMonth`：必填，格式 `YYYY-MM`
- `employeeId`：可选
- `salaryType`：可选
- `calculationStatus`：可选
- `reviewStatus`：可选
- `page`：可选
- `pageSize`：可选

### 响应

返回分页后的 `MonthlyPayrollResult` 列表，并附带员工名称、员工编号。

---

## 5.2 获取薪酬结果详情

### 请求

```http
GET /api/admin/payroll-results/:id
```

### 响应

返回完整 `MonthlyPayrollResult`，并建议附带：

- 员工基础信息
- 薪资配置快照
- 月度考勤汇总
- 一次性薪资项明细

---

## 5.3 生成单个员工薪酬


### 请求体

```ts
interface GenerateOnePayrollRequest {
  employeeId: string | number;

}
```

### 响应

返回 `MonthlyPayrollResult`。

---

## 5.4 批量生成月份薪酬

### 请求

```http
POST /api/admin/payroll-results/generate-monthly
```

### 请求体

```ts
interface GenerateMonthlyPayrollRequest {
  yearMonth: string;
  employeeIds?: Array<string | number>;
}
```

### 处理

- 为指定员工或全部符合条件员工生成薪酬结果。
- 已存在结果时默认跳过，避免覆盖人工核对中的结果。
- 如需覆盖，应使用重新计算接口。

---

## 5.5 重新计算单个员工薪酬

### 请求

```http
POST /api/admin/payroll-results/recalculate-one
```

### 请求体

```ts
interface RecalculateOnePayrollRequest {
  employeeId: string | number;
  yearMonth: string;
}
```

### 处理

- 重新读取薪资配置、考勤汇总、一次性薪资项。
- 更新 `MonthlyPayrollResult`。
- 如果结果已确认，是否允许重算需后续确认；本轮建议禁止重算 confirmed 结果。

---

## 5.11 标记核对通过

### 请求

```http
PATCH /api/admin/payroll-results/:id/approve
```

### 处理

- 设置 `reviewStatus = approved`。
- 不修改金额字段。

---

## 5.12 驳回薪酬结果

### 请求

```http
PATCH /api/admin/payroll-results/:id/reject
```

### 请求体

```ts
interface RejectPayrollResultRequest {
  reason?: string;
}
```

### 处理

- 设置 `reviewStatus = rejected`。
- 本轮可暂不持久化驳回原因；如需要，后续增加字段。

---

## 5.13 确认薪酬结果

### 请求

```http
PATCH /api/admin/payroll-results/:id/confirm
```

### 处理

确认前校验：

- `calculationStatus = calculated`
- `reviewStatus = approved`
- 不是阻断结果

确认后：

- 设置 `calculationStatus = confirmed`
- 写入 `confirmedAt`

## 6. 校验规则

### 6.1 SalaryProfile 校验

- `employeeId` 必须存在。
- `salaryType` 必须为合法枚举。
- `salaryType = fixed` 时，`fixedSalary` 必填且大于等于 0。
- `salaryType = hourly` 时，`hourlyRate` 必填且大于等于 0。
- 固定工资和时薪不能同时参与计算。
- `currency` 必须为合法枚举。

### 6.2 SalaryAdjustmentItem 校验

- `employeeId` 必须存在。
- `yearMonth` 必须为 `YYYY-MM`。
- `type` 必须为合法枚举。
- `name` 去除首尾空格后不能为空。
- `amount` 必须大于等于 0。

### 6.3 薪酬结果确认校验

- 阻断结果不能确认。
- 未核对通过不能确认。
- 已确认结果不允许重复确认。

## 7. 与其他模块关系

### 7.1 与员工管理模块

读取：

- 员工 ID
- 员工编号
- 员工姓名
- 员工状态
- 薪资基础配置

不修改员工档案。

### 7.2 与考勤计算模块

读取：

- `MonthlyAttendanceSummary.yearMonth`
- `MonthlyAttendanceSummary.totalValidHours`
- `MonthlyAttendanceSummary.totalStandardHours`
- `MonthlyAttendanceSummary.totalOvertimePayHours`
- `MonthlyAttendanceSummary.canGeneratePayroll`
- `MonthlyAttendanceSummary.blockedReason`

不计算每日考勤。

### 7.3 与考勤规则管理模块

薪酬模块不直接编辑考勤规则，也不重新匹配规则。

加班相关输入应来自考勤计算模块的月度汇总或已固化结果。

## 8. 数据迁移建议

当前代码只有前端即时计算结果，没有独立后端薪酬模型。

迁移建议：

1. 将现有 `Employee.hourlyRate` 和 `Employee.currency` 迁移到 `SalaryProfile`。
2. 默认将现有员工设置为 `salaryType = hourly`。
3. `fixedSalary` 初始为空。
4. 创建空的一次性薪资项表。
5. 基于已有考勤月度汇总生成 `MonthlyPayrollResult`。
6. 不迁移 `taxRate` 为核心字段；如需要保留历史效果，可为对应月份生成一条扣款项。

---

## 9. 错误码建议

- `PAYROLL_RESULT_NOT_FOUND`：薪酬结果不存在
- `SALARY_PROFILE_NOT_FOUND`：薪资配置不存在
- `SALARY_PROFILE_INVALID`：薪资配置不合法
- `SALARY_ADJUSTMENT_ITEM_NOT_FOUND`：一次性薪资项不存在
- `MONTHLY_ATTENDANCE_SUMMARY_NOT_FOUND`：月度考勤汇总不存在
- `MONTHLY_ATTENDANCE_SUMMARY_BLOCKED`：月度考勤汇总不可进入薪资核算
- `PAYROLL_RESULT_BLOCKED`：薪酬结果被阻断
- `PAYROLL_RESULT_NOT_APPROVED`：薪酬结果未核对通过
- `PAYROLL_RESULT_ALREADY_CONFIRMED`：薪酬结果已确认
- `YEARMONTH_INVALID`：年月格式错误

---

## 10. 验收标准

后端完成后需要满足：

- 可以维护员工薪资基础配置。
- 可以维护一次性薪资项。
- 可以生成单个员工某月薪酬结果。
- 可以批量生成某月薪酬结果。
- 可以重新计算薪酬结果。
- 薪酬计算读取月度考勤汇总，不直接读取原始考勤记录。
- 支持固定工资和时薪二选一。
- 支持补贴、扣款、其他一次性薪资项。
- 支持计算应发金额、扣款金额、实发金额。
- 可以标记核对通过、驳回、确认薪酬。
- 阻断结果有明确阻断原因。
- 不把全局 `taxRate` 作为核心薪酬字段。
