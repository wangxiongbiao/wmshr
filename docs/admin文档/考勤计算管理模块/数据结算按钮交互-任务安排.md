# 数据结算按钮交互 - 任务安排

- 确认时间：2026-06-05 23:57:25 PDT
- 范围：仅梳理并实施考勤模块“数据结算”按钮交互，不扩展到其他按钮、薪资模块或定时任务底层逻辑。
- 已确认业务规则：
  - 按钮文案为“数据结算”。
  - 点击后需要二次确认。
  - 当前页面筛选条件可以存在，但不影响执行范围。
  - 执行范围固定为：结算前一天考勤，并生成今天考勤底稿。
  - 执行结果只提示“结算成功”。
  - 执行中需要防重复点击。
  - tooltip 只解释功能。

## 推荐实施顺序

### 任务 1：固化按钮业务定义

- 目标：明确“数据结算”按钮代表唯一业务动作。
- 输入：已确认规则：固定执行“结算前一天 + 生成今天底稿”，且不受当前筛选条件影响。
- 输出：按钮业务定义与实现边界。
- 验收标准：点击按钮不会使用当前筛选日期/员工决定执行范围。
- 依赖关系：无。
- 执行方式：HITL 已确认。

### 任务 2：实现点击前确认弹窗

- 目标：点击“数据结算”后先确认，再执行接口。
- 输入：按钮业务定义。
- 输出：确认弹窗标题、说明、取消按钮、确认按钮。
- 建议文案：
  - 标题：确认数据结算
  - 内容：系统将结算前一天考勤，并生成今天考勤底稿。此操作不受当前筛选条件影响，是否继续？
  - 按钮：取消 / 确认结算
- 验收标准：点击按钮不会立即执行；确认后才执行。
- 依赖关系：任务 1。
- 执行方式：AFK 实施，HITL 验收。

### 任务 3：实现执行中状态和防重复点击

- 目标：用户确认后展示处理中状态，并阻止重复提交。
- 输入：提交状态变量、现有数据结算接口调用。
- 输出：loading / disabled 状态、执行中文案。
- 验收标准：接口执行中不能重复点击“数据结算”。
- 依赖关系：任务 2。
- 执行方式：AFK。

### 任务 4：调整 tooltip 文案

- 目标：tooltip 只解释按钮功能。
- 输入：用户确认的 tooltip 约束。
- 输出：简短 tooltip 文案。
- 建议文案：结算前一天考勤，并生成今天考勤底稿。
- 验收标准：tooltip 不包含额外业务建议或复杂说明。
- 依赖关系：任务 1。
- 执行方式：AFK 实施，HITL 验收。

### 任务 5：实现成功结果轻量提示

- 目标：接口完成后只提示“结算成功”。
- 输入：每日维护接口返回结果。
- 输出：成功提示内容。
- 展示字段：仅展示“结算成功”。
- 验收标准：用户能直接看到结算已完成，不展示底层日期、底稿数量或失败统计。
- 依赖关系：任务 3。
- 执行方式：AFK 实施，HITL 验收。

### 任务 6：实现失败和部分失败反馈

- 目标：失败时保持轻量反馈，不引入复杂流程。
- 输入：接口错误。
- 输出：整体失败提示。
- 验收标准：整体失败显示错误；接口成功时只显示“结算成功”。
- 依赖关系：任务 5。
- 执行方式：AFK。

### 任务 7：实现完成后刷新规则

- 目标：执行完成后刷新当前考勤列表，同时保留当前筛选上下文。
- 输入：当前列表加载函数、筛选状态。
- 输出：刷新后的列表。
- 验收标准：页面显示最新结果，不清空当前筛选条件。
- 依赖关系：任务 5、任务 6。
- 执行方式：AFK。

### 任务 8：验证与回填

- 目标：验证按钮交互改动没有破坏类型检查和构建，并回填进度。
- 输入：修改后的前端代码和任务进度文档。
- 输出：验证结果、进度更新。
- 验收标准：至少通过 admin 类型检查；如条件允许，通过 admin build。
- 依赖关系：任务 1-7。
- 执行方式：AFK，最终 HITL 验收。
