# 员工管理模块前端开发设计文档

## 1. 文档目标

本文档用于指导 `wmshr-admin` 员工管理模块的前端开发改造。

设计原则：

- 以现有代码字段和页面结构为主。
- 现有代码缺失但业务文档已确认的字段，从业务文档补齐。
- 员工管理只负责员工主数据、当前考勤规则关联、规则历史展示、计薪基础信息维护。
- 不在员工管理模块中实现考勤规则配置、考勤计算、薪资核算。

关联文档：

- `wmshr-admin-business-flow-data-relations.md`
- `wmshr-admin-field-gap-analysis.md`
- `admin文档/员工原理模块/员工管理模块-后端开发设计文档.md`

## 2. 已确认业务规则

### 2.1 员工编号

- 员工编号由后端自动生成。
- 前端新增员工时不允许手动输入员工编号。
- 编辑员工时员工编号只读展示。

### 2.2 联系方式

- 联系方式只保留手机号。
- 不设计邮箱、紧急联系人等字段。

### 2.3 员工状态

员工状态采用：

- 在职：`active`
- 休假：`on_leave`
- 试用：`probation`
- 停用：`disabled`
- 离职：`resigned`

### 2.4 历史数据

- 停用和离职员工继续出现在历史考勤和历史薪资中。
- 停用和离职员工默认不参与新的考勤和薪资核算。

### 2.5 删除规则

- 员工删除为停用/软删除。
- 前端不再出现“永久删除员工档案，无法恢复”的文案。

### 2.6 计薪方式

- 固定工资和时薪只能二选一。
- 选择固定工资时只填写固定工资金额。
- 选择时薪时只填写时薪金额。

## 4. 前端目标数据结构

### 4.1 EmployeeStatus

```ts
export type EmployeeStatus =
  | 'active'
  | 'on_leave'
  | 'probation'
  | 'disabled'
  | 'resigned';
```

### 4.2 SalaryType

```ts
export type SalaryType = 'fixed' | 'hourly';
```

### 4.3 Employee

```ts
export interface Employee {
  id: number;
  employeeNo: string;
  name: string;
  gender: 'male' | 'female';
  country: 'MM' | 'TH' | 'CN' | 'VN' | 'KH';

  role: string;
  dept: string;

  fixedSalary?: number;
  currency: 'THB' | 'USD' | 'MYR' | 'IDR';

}
```

export interface EmployeeAttendanceRuleHistory {

## 6. 员工列表页设计

### 6.1 默认展示范围

默认展示：

- 在职员工
- 试用员工
- 休假员工

默认不展示：

- 停用员工
- 离职员工

需要提供筛选项允许显示：

- 停用员工
- 离职员工

### 6.2 员工卡片展示字段

员工卡片建议展示：

- 头像
- 姓名
- 员工编号
- 手机号
- 国家
- 性别
- 员工状态
- 岗位
- 部门/区域
- 当前考勤规则名称
- 计薪方式
- 固定工资或时薪金额
- 币种
- 入职日期

### 6.3 计薪展示规则

当 `salaryType = hourly`：

```text
计薪方式：时薪
时薪：฿280 / 小时
```

当 `salaryType = fixed`：

```text

```

### 6.4 状态展示规则

状态标签建议：

- 在职：绿色
- 试用：蓝色
- 休假：黄色或浅蓝

## 8. 新增员工弹窗设计

### 8.1 表单字段

#### 基础信息

- 头像
- 员工编号，只读，显示“系统自动生成”
- 姓名
- 性别
- 国家/地区
- 手机号
- 岗位
- 部门/区域
- 入职日期
- 员工状态

#### 考勤规则

- 当前考勤规则
- 规则生效日期

#### 薪资信息

- 计薪方式
- 固定工资金额
- 时薪金额
- 币种

### 8.2 默认值建议

- 性别：`female`
- 国家：`MM`
- 员工状态：`active`
- 入职日期：当天
- 规则生效日期：默认等于入职日期
- 计薪方式：`hourly`
- 币种：`THB`

### 8.3 交互规则

#### 员工编号

- 新增时不允许输入。
- 显示“系统自动生成”。
- 创建成功后由接口返回并展示。

#### 当前考勤规则

- 必选。
- 下拉选择。
- 选项来自考勤规则列表。
- 只显示可用规则。

#### 规则生效日期

- 必填。
- 默认等于入职日期。
- 用于创建员工的第一条规则历史。

#### 计薪方式互斥

选择固定工资时：

- 显示固定工资金额。
- 固定工资金额必填。
- 隐藏或禁用时薪金额。

选择时薪时：

- 显示时薪金额。
- 时薪金额必填。
- 隐藏或禁用固定工资金额。

## 9. 编辑员工弹窗设计

### 9.1 可编辑字段

- 头像
- 姓名
- 性别
- 国家/地区
- 手机号
- 岗位
- 部门/区域
- 入职日期
- 员工状态
- 当前考勤规则
- 规则生效日期，仅修改规则时必填
- 计薪方式
- 固定工资金额
- 时薪金额
- 币种

### 9.2 只读字段

- 员工 ID
- 员工编号

### 9.3 修改考勤规则

当用户修改 `attendanceRuleId`：

- 必须显示并填写新规则生效日期。
- 保存前提示：

```text
修改考勤规则会生成新的规则历史，不会影响历史考勤和历史薪资。
```

提交时需要包含：

- `attendanceRuleId`
- `ruleEffectiveStartDate`

### 9.4 修改计薪方式

从时薪切换到固定工资：

- 固定工资金额必填。
- 时薪金额不再生效。

从固定工资切换到时薪：

- 时薪金额必填。
- 固定工资金额不再生效。

## 10. 停用/离职交互设计

### 10.1 删除按钮语义调整

当前“删除员工”应调整为：

- 停用员工
- 标记离职

不再使用永久删除文案。

### 10.2 停用确认文案

```text
确认停用该员工？
停用后，该员工不会参与新的考勤和薪资核算，但历史考勤、历史薪资和规则历史会继续保留。
```

### 10.3 离职确认文案

```text
确认将该员工标记为离职？
离职后，该员工不会参与新的考勤和薪资核算，但仍可在历史考勤和历史薪资中查询。
```

### 10.4 停用/离职后行为

- 默认员工列表隐藏停用/离职员工。
- 勾选包含停用/离职后展示。
- 历史考勤和历史薪资页面仍可展示这些员工。

## 12. 前端校验规则

### 12.1 基础校验

- 姓名必填。
- 手机号必填。
- 性别必填。
- 国家/地区必填。
- 岗位必填。
- 部门/区域建议必填。
- 入职日期必填。
- 员工状态必填。
- 币种必填。

### 12.2 考勤规则校验

- 当前考勤规则必选。
- 新增员工时规则生效日期必填。
- 修改考勤规则时规则生效日期必填。

### 12.3 薪资字段校验

- 计薪方式必选。
- 固定工资和时薪互斥。
- `salaryType = fixed` 时固定工资金额必填且不能小于 0。
- `salaryType = hourly` 时时薪金额必填且不能小于 0。

### 12.4 状态校验

- 停用/离职员工不应出现在新考勤和新薪资默认选择范围中。
- 停用/离职员工仍可在历史记录中展示。

## 14. 验收标准

- 员工列表能展示员工编号、手机号、当前规则、计薪方式。
- 新增员工时员工编号显示为系统自动生成，不可手动输入。
- 编辑员工时员工编号只读。
- 手机号可录入并校验必填。
- 员工状态包含在职、休假、试用、停用、离职。
- 停用/离职员工默认不参与新考勤和新薪资。
- 停用/离职员工仍可在历史记录中展示。
- 删除交互改为停用/离职，不再提示永久删除。
- 固定工资和时薪互斥。
- 修改考勤规则时提示会生成规则历史。
- 员工规则历史可展示。
