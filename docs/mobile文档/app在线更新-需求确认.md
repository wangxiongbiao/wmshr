# App 在线更新 - 需求确认

## 目标
- 为 Android App 增加一个最简单可用的在线更新能力。
- 后端提供一个最新版本接口，字段仅包含：版本号、版本内容、版本 URL。
- App 本地版本号与线上版本号不一致时，全局弹出更新提醒；用户可以选择立即更新或暂不更新。
- 更新 URL 为 APK 直链，点击更新后走下载并自动拉起安装流程。

## 已确认范围
- 平台范围：只考虑 Android。
- 版本比较规则：本地版本号与线上版本号只要不一致，就提示更新。
- 接口字段范围：`version`、`content`、`url`。
- 更新提示：用户可自行选择立即更新或暂不更新。
- 更新内容展示：纯文本即可，不做富文本、多语言扩展。
- 更新 URL：直接 APK 安装包直链，不做中转下载页。
- 下载行为目标：下载 APK，并自动拉起系统安装流程。

## 明确不做
- 不做 iOS 更新能力。
- 不做多版本历史列表。
- 不做灰度发布、渠道分流、最低支持版本、增量包等扩展能力。
- 不做复杂版本比较策略；当前只按“不一致即更新”。
- 不擅自扩展后台配置范围，除非实施阶段为当前最小方案必需。

## 当前待实施口径
- 后端提供一个“最新 Android 版本信息”接口。
- App 在全局启动/进入主流程前检查版本。
- 一旦发现线上版本与本地版本不一致，立即弹出全局更新提示。
- 用户点击更新后触发下载，并拉起系统安装；如果暂不更新，可先继续使用 App。
