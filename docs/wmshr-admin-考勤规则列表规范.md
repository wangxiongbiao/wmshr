# WMshr Admin 考勤规则列表规范

## 1. 目标

考勤规则列表页用于在 admin 后台中快速完成以下任务：

- 查看规则核心配置
- 按规则名称、启用状态、有效期状态筛选规则
- 新增规则
- 编辑规则
- 启用或停用规则
- 查看规则关联员工
- 在保持卡片结构的前提下支持分页浏览

当前页面以“卡片式规则配置列表”为基准，不使用表格替代。

---

## 2. 页面结构

考勤规则列表页由 3 个区域组成：

1. 顶部筛选区
2. 中部规则卡片列表区
3. 底部分页区

页面整体保持当前 admin 主题风格：

- 使用 `glass-panel` 作为主容器基底
- 规则卡片沿用当前圆角、边框、阴影系统
- 主操作按钮使用品牌蓝 `brand`
- 状态标签沿用当前状态色系统

---

## 3. 顶部筛选区规范

### 3.1 布局

筛选区必须尽量紧凑，但不能因为压缩布局导致文案和操作混在一起。

当前建议布局：

- 左侧：搜索规则、启用状态、有效期状态
- 右侧：操作区

规则：

- 当筛选项只有 `3` 个时，桌面端优先保持为单行布局
- 当视口不足时允许自然换行
- 不允许出现“标题文案 + 状态文案”挤在同一视觉层级、造成阅读混乱

### 3.2 字段

筛选项包括：

- 搜索规则
- 启用状态
- 有效期状态

### 3.3 搜索范围

搜索关键词匹配以下字段：

- 规则名称
- 上班时间
- 下班时间
- 休息开始时间
- 休息结束时间

### 3.4 操作区规范

右侧区域标题固定为：

- `操作`

右侧区域内容包括：

- `重置`
- `新增`

规则：

- 默认状态下不显示 `默认视图` 这类占位文案
- 只有在真的启用了筛选条件时，才显示轻量提示，例如 `筛选中 2`
- `新增` 为主按钮，始终保持明显视觉优先级
- `重置` 为次按钮，视觉弱于主按钮

### 3.5 交互规则

- 任一筛选项变化后，回到第一页
- `重置` 恢复所有默认值
- 搜索和筛选变化后只影响当前列表结果，不改变卡片结构

---

## 4. 规则卡片规范

### 4.1 卡片布局

必须保留当前卡片式布局，不改为大横条信息墙，也不改成表格。

每张卡片包含：

- 规则名称
- 启用状态
- 有效期状态
- 生效日期范围
- 班次时间
- 休息时间
- 标准工时
- 关联员工数
- 加班规则
- 加班费标准
- 加班计量说明

补充规则：

- `标准工时` 不是人工输入字段
- `标准工时` 必须根据 `上班时间 - 下班时间 - 休息时间` 自动计算
- 新增和编辑规则时，只展示自动计算结果，不提供手动填写输入框

### 4.2 操作按钮

卡片右上角操作区包括：

- 查看关联员工
- 编辑
- 启用 / 停用

规则：

- 启用中的规则显示 `停用`
- 已停用规则显示 `启用`
- 启用/停用按钮颜色要和动作语义一致

### 4.3 状态标签

规则状态标签沿用当前颜色语义：

- 启用：绿色倾向
- 停用：中性/灰色倾向
- 生效中：绿色倾向
- 未开始：蓝色倾向
- 已过期：警告或风险倾向

---

## 5. 分页规范

### 5.1 模式

当前规则列表使用分页浏览。

分页参数包括：

- `page`
- `pageSize`

当前默认：

- `pageSize = 6`

### 5.2 分页区布局

底部分页区包含：

- 当前页 / 总页数
- 当前显示区间
- 总规则数
- 上一页按钮
- 页码按钮
- 下一页按钮

### 5.3 页码规则

- 页数较少时显示全部页码
- 页数较多时使用省略号
- 当前页必须高亮
- 页码按钮必须可直接点击跳转

---

## 6. 加载与空状态规范

### 6.1 列表加载

规则列表首次加载时，可显示列表级 loading。

但以下场景不能让整个列表进入空白 loading：

- 点击编辑规则
- 打开关联员工弹窗
- 打开启用/停用确认弹窗

这类场景应优先保留当前列表，只在局部或弹窗层处理加载状态。

补充规则：

- 点击 `编辑` 时，页面应显示全局遮罩 loading
- 等规则详情返回后，再打开编辑弹窗
- 不允许因为进入编辑弹窗而让规则列表本身切成整块 loading

### 6.2 空状态

当当前筛选条件下没有规则时，显示：

- 明确的空状态文案
- 不破坏顶部筛选区
- 不隐藏新增规则入口

建议文案：

`当前筛选条件下没有考勤规则`

---

## 7. 规则弹窗联动规范

### 7.1 编辑弹窗

考勤规则编辑弹窗遵循三段式结构：

1. Header
2. Body
3. Footer

规则：

- 仅 Body 可滚动
- Footer 固定在底部
- Footer 包含取消、保存规则按钮
- 用户不需要滚动到底部才能保存
- 打开编辑前如需加载详情，应使用全局遮罩 loading，而不是列表 loading
- `标准工时` 在弹窗内展示为自动计算结果，而不是可编辑输入项

### 7.2 轻确认弹窗

启用 / 停用规则属于轻确认交互：

- 不允许使用浏览器原生确认框
- 必须复用统一弹窗规范
- 文案要明确说明对员工选择和历史考勤的影响

---

## 8. 禁止事项

以下实现不允许再出现：

- 搜索区右侧显示 `默认视图`
- `操作` 标题和状态说明文案挤在一起
- 中间再插入说明型卡片，例如“考勤规则卡片列表”
- 将规则卡片改成表格
- 使用浏览器原生 `alert` / `confirm` / `prompt`
