# Project Issues

### 2026-07-02 问题 1：Play Developer API 权限检查被 Python CA 证书链拦截
- 原因：本机 Homebrew Python 3.14 通过 `urllib.request` 请求 `https://oauth2.googleapis.com/token` 时，TLS 校验报错：`SSL: CERTIFICATE_VERIFY_FAILED` / `unable to get local issuer certificate`。
- 导致的问题：首次用 Python 标准库直接调用 Google OAuth + Android Publisher API 的权限检查未能到达 Google 服务端，不能据此判断服务账号是否有 Play Console 上传权限。
- 原计划验证入口：使用 `apps/mobile/google-service-account.json` 生成 service-account JWT，换取 `androidpublisher` access token，再对 `com.wmshr.app` 创建临时 edit 并立即删除，只验证权限、不上传 AAB。
- 当前判断：这是本机 Python CA 环境问题，不是 Google Play 权限或 AAB 本身的结论。
- 已尝试：服务账号 JSON 结构检查通过；AAB 文件存在；Python 直连 OAuth token endpoint 被 CA 校验拦截。
- 下一步：改用系统 `curl` 调用 OAuth / Android Publisher API，继续完成同样的临时 edit 权限验证。
- 最终结果：系统 `curl` 路径可正常到达 Google OAuth，服务账号 token 获取成功；但对 `com.wmshr.app` 调用 Android Publisher `edits.insert` 返回 `403 PERMISSION_DENIED` / `The caller does not have permission`。因此当前服务账号尚未具备该 Play Console 应用的发布/edit 权限，暂时不能通过 API 替用户上传 AAB。
- 后续处理：需要在 Google Play Console 中把该 service account 加为用户，授予 `com.wmshr.app` 对应应用的发布版本管理权限；之后再重试临时 edit 权限验证。
- 追加复验（2026-07-02）：本轮重新用 `apps/mobile/google-service-account.json` 走系统 `curl` + Android Publisher API 做只创建临时 edit 的权限检查；OAuth token 获取成功，但 `edits.insert` 仍返回 `403 PERMISSION_DENIED` / `The caller does not have permission`，因此当前仍不能通过 API 代替用户上传到 Google Play Console。
