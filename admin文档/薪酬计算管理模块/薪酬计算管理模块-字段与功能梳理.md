## 2. 模块定位

薪酬计算管理模块负责按自然月生成员工薪酬结果，并展示薪酬明细，便于人工核对。

模块负责：

- 读取员工计薪方式。
- 读取员工薪资基础配置。
- 读取自然月考勤汇总。
- 读取当前自然月一次性薪资项。
- 计算固定工资或时薪工资。
- 计算加班费。
- 汇总补贴。
- 汇总扣款。
- 汇总其他项。
- 生成自然月薪酬结果。
- 展示薪酬明细和金额来源。
- 标记计算状态和核对状态。

模块不负责：

- 员工基础档案维护。
- 员工考勤规则配置。
- 每日考勤计算。
- 原始考勤记录录入。
- 修改历史考勤规则。

```text

```

### 3.2 当前薪资结果字段

当前代码中已有 `PayrollSummary`：

```ts
export interface PayrollSummary {
  emp: Employee;
  valid: number;
  ot: number;
  basePay: number;
  otPay: number;
  gross: number;
  net: number;
}
```

字段含义：

- `emp`：员工信息。
- `valid`：有效总工时，当前从原始考勤记录即时计算累加。
- `ot`：加班总时长，当前从原始考勤记录即时计算累加。
- `basePay`：基础薪资。
- `otPay`：加班费用。
- `gross`：应发金额，当前等于基础薪资 + 加班费用。
- `net`：税后实发，当前等于应发金额扣除税率。

### 3.3 当前薪资计算逻辑

当前 `PayrollTable.tsx` 中直接遍历原始考勤记录并即时计算薪资：

```ts
let valid = 0, ot = 0, otPay = 0;
attendance.filter(r => r.empId === emp.id).forEach(rec => {
  const d = calcAttendanceDetails(rec, config);
  valid += d.valid;
  ot += d.ot;
  otPay += d.ot * config.otHourlyFee;
});

const basePay = (valid - ot) * emp.hourlyRate;
const gross = basePay + otPay;
const net = gross * (1 - config.taxRate);
```

当前代码实际口径：

```text
基础薪资 = 标准内有效工时 × 员工时薪
加班费用 = 加班时长 × 全局加班费标准
应发金额 = 基础薪资 + 加班费用
实发金额 = 应发金额 × (1 - 全局税率)
```

## 8. 计算规则梳理

### 8.1 前置条件

薪酬计算前需要确认：

- 员工存在。
- 员工有有效薪资基础配置。
- 员工薪资配置中固定工资和时薪二选一。
- 自然月考勤汇总已生成。
- 自然月考勤汇总允许进入薪资核算。
- 当前自然月一次性薪资项已读取。

### 8.2 固定工资员工

计算方式：

```text
基础工资 = fixedSalary
```

说明：

- 固定工资员工按自然月固定金额进入核算。
- 加班费是否仍然叠加，当前业务文档允许加班费进入薪资结果，因此本轮按叠加处理。

### 8.3 时薪员工

当前代码口径：

```text
基础薪资 = (有效总工时 - 加班总时长) × 时薪
```

业务文档口径：

```text
时薪工资 = 自然月有效工时 × 时薪
```

两者存在差异。

本轮建议薪酬计算文档采用更不重复计薪的口径：

```text
时薪工资 = 标准内有效工时 × 时薪
标准内有效工时 = max(0, 有效工时 - 加班计薪时长)
```

原因：

- 当前系统已经单独计算加班费。
- 如果时薪工资用全部有效工时，同时再叠加加班费，可能重复计算加班时间。
- 当前代码实际也采用“标准内有效工时工资 + 加班费”的方向。

需要 HITL 确认：如果业务希望“全部有效工时 × 时薪 + 加班补差”，应后续单独调整口径。

### 8.4 加班费

计算方式：

```text
加班费 = 自然月加班计薪时长 × 加班费标准
```

来源：

- 自然月加班计薪时长来自 `MonthlyAttendanceSummary.totalOvertimePayHours`。
- 加班费标准来自考勤规则计算时固化的规则，或由考勤计算结果汇总后的加班费字段提供。

注意：

- 不使用 `overtimeMultiplier`。
- 加班时长必须使用考勤计算后的计薪时长，不使用原始加班时长。

### 8.5 补贴合计

计算方式：

```text
补贴合计 = 当前员工当前自然月 type = allowance 的 SalaryAdjustmentItem 金额合计
```

### 8.6 扣款合计

计算方式：

```text
扣款合计 = 当前员工当前自然月 type = deduction 的 SalaryAdjustmentItem 金额合计
```

### 8.7 其他合计

计算方式：

```text
其他合计 = 当前员工当前自然月 type = other 的 SalaryAdjustmentItem 金额合计
```

说明：

- 当前先按正向加项处理。
- 如需同时支持正负方向，后续应增加 `direction` 字段。

### 8.8 应发金额

建议计算：

```text
应发金额 = 基础工资或时薪工资 + 加班费 + 补贴合计 + 其他合计
```

### 8.9 扣款金额

建议计算：

```text
扣款金额 = 扣款合计
```

当前代码中的 `taxRate` 不作为本轮核心薪酬字段。

处理建议：

- 如果确实需要扣税，先作为 `SalaryAdjustmentItem` 的扣款项录入。
- 不继续使用全局 `taxRate` 自动扣税，除非后续业务明确税率规则。

### 8.10 实发金额

建议计算：

```text
实发金额 = 应发金额 - 扣款金额
```

## 9.6 阻断处理

阻断场景：

- 员工薪资配置缺失。
- 员工薪资配置固定工资和时薪同时存在。
- 员工薪资配置固定工资和时薪都为空。
- 月度考勤汇总缺失。
- 月度考勤汇总不可进入薪资核算。
- 币种缺失。

阻断结果：

- 生成 `calculationStatus = blocked` 的薪酬结果。
- 展示 `blockedReason`。
- 不允许确认薪酬。

## 12. 当前字段冲突和待确认点

### 12.1 当前只支持时薪

当前代码所有员工都有 `hourlyRate`，没有固定工资。

后续需要改为：

```text
salaryType = fixed 时只使用 fixedSalary
salaryType = hourly 时只使用 hourlyRate
```

### 12.2 当前基础薪资口径与业务文档描述存在差异

当前代码：

```text
基础薪资 = 标准内有效工时 × 时薪
```

业务文档原描述：

```text
时薪员工按自然月有效工时 × 时薪计算
```

本轮建议采用当前代码方向，避免加班重复计薪，但该点需要人工确认。

### 12.3 当前税率没有业务确认

当前代码：

```text
net = gross × (1 - taxRate)
```

业务文档没有确认税率规则。

本轮建议：

- 不把 `taxRate` 作为薪酬计算核心字段。
- 如需税费，作为扣款项录入。

### 12.4 当前薪资直接读取原始考勤记录

当前代码直接遍历 `AttendanceRecord[]` 计算薪资。

后续应改为：

```text
薪酬计算读取 MonthlyAttendanceSummary，不直接读取原始考勤记录。
```
