# 考勤规则管理模块前端开发设计文档

## 1. 文档目标

本文档用于指导 `wmshr-admin` 考勤规则管理模块的前端开发。

设计原则：

- 以现有 `AppConfig` 中已经存在的考勤配置字段为基础。
- 现有代码缺失但业务流程文档已定义的字段，从业务文档补齐。
- 考勤规则管理只负责规则维护、启用停用、关联员工查看。
- 不在考勤规则管理模块中录入员工考勤、不计算工资、不修改员工规则关联。

关联文档：

- `wmshr-admin-business-flow-data-relations.md`
- `wmshr-admin-field-gap-analysis.md`
- `admin文档/员工原理模块/员工管理模块-前端开发设计文档.md`
- `admin文档/考勤规则管理模块/考勤规则管理模块-后端开发设计文档.md`

## 2. 已确认业务规则

### 2.1 被员工引用后允许直接编辑

- 一条考勤规则即使已经被员工引用，也允许直接编辑。
- 前端不做“已引用规则禁止编辑”的限制。
- 保存前需要提示影响范围。
- 修改后，关联员工后续考勤计算按新规则执行。
- 如果历史考勤重新计算，历史结果也可能因规则变化而改变。

保存提示文案：

```text
该考勤规则已被员工引用。保存修改后，关联员工后续考勤计算将按新规则执行。如重新计算历史考勤，历史结果也可能变化。
```

### 2.2 支持查看关联员工弹窗

- 考勤规则列表和规则详情中提供“查看关联员工”入口。
- 弹窗只查看，不在弹窗里切换员工规则。
- 员工切换规则仍由员工管理模块负责。

### 2.3 停用后不强制员工更换规则

- 停用规则只影响新关联。
- 已关联该规则的员工不需要强制更换规则。
- 已关联员工仍可继续使用该规则参与考勤计算。
- 历史考勤和历史薪资仍可追溯该规则。

停用提示文案：

```text
停用后，该规则不再允许新员工选择或员工新切换到该规则，但已关联员工可继续使用，历史考勤不受影响。
```

### 2.4 暂不设计迟到、早退、缺勤规则

本轮不做：

- 迟到规则
- 早退规则
- 缺勤规则

前端不展示这些配置项，计算逻辑也不依赖这些字段。

### 2.5 生效结束时间允许为空

- 生效开始时间必填。
- 生效结束时间可为空。
- 生效结束时间为空表示长期有效。
- 如果填写生效结束时间，则不能早于生效开始时间。

## 3. 当前前端现状

当前代码没有独立的 `AttendanceRule` 模型，只有全局 `AppConfig`。

当前已有字段：

```ts
export interface AppConfig {
  startShift: string;
  endShift: string;
  breakStart: string;
  breakEnd: string;
  standardHours: number;
  otHourlyFee: number;
  overtimeMultiplier: number;
  taxRate: number;
  dailyBreakMinutes: number;
  currency: CurrencyCode;
}
```

当前默认值：

```ts
export const INITIAL_CONFIG: AppConfig = {
  startShift: "08:30",
  endShift: "17:30",
  breakStart: "12:00",
  breakEnd: "13:00",
  standardHours: 8,
  otHourlyFee: 50,
  overtimeMultiplier: 1.5,
  taxRate: 0.05,
  dailyBreakMinutes: 60,
  currency: 'THB'
};
```

当前设置弹窗实际展示：

- 上班时间
- 下班时间
- 午休开始
- 午休结束
- 加班费标准

## 4. 前端目标数据结构

### 4.1 AttendanceRule

```ts
export type OvertimeRounding = 'floor_to_half_hour';

export interface AttendanceRule {
  id: string | number;
  name: string;
  isActive: boolean;

  effectiveStartDate: string;
  effectiveEndDate: string | null;

  startShift: string;
  endShift: string;
  breakStart: string;
  breakEnd: string;
  standardHours: number;

  overtimeEnabled: boolean;
  otHourlyFee: number;
  overtimeMinUnitHours: 0.5;
  overtimeRounding: OvertimeRounding;

  createdAt?: string;
  updatedAt?: string;
}
```

### 4.2 AttendanceRuleRelatedEmployee

用于查看关联员工弹窗。

```ts
export interface AttendanceRuleRelatedEmployee {
  id: string | number;
  employeeNo: string;
  name: string;
  dept: string;
  role: string;
  status: 'active' | 'on_leave' | 'probation' | 'disabled' | 'resigned';
  relationStartDate: string;
  relationEndDate: string | null;
  isCurrentRelation: boolean;
}
```

### 4.3 AttendanceRuleFormData

前端表单可以和主模型保持接近，但不包含系统字段。

```ts
export interface AttendanceRuleFormData {
  name: string;
  isActive: boolean;
  effectiveStartDate: string;
  effectiveEndDate: string | null;
  startShift: string;
  endShift: string;
  breakStart: string;
  breakEnd: string;
  standardHours: number;
  overtimeEnabled: boolean;
  otHourlyFee: number;
}
```

## 5. 字段处理规则

### 5.1 从现有 AppConfig 迁移到 AttendanceRule

需要迁移：

- `startShift`
- `endShift`
- `breakStart`
- `breakEnd`
- `standardHours`
- `otHourlyFee`

### 5.2 不进入考勤规则表单的现有字段

#### overtimeMultiplier

- 当前代码已有，但业务已确认按固定小时加班费计算。
- 不进入考勤规则表单。
- 后续如清理代码，应从实际计算和展示中移除或标记为废弃。

#### taxRate

- 属于薪资/扣款口径，不属于考勤规则管理。
- 不进入本模块。

#### dailyBreakMinutes

- 与 `breakStart` / `breakEnd` 重复。
- 前端可根据休息开始和结束时间派生展示。
- 不作为独立输入项。

#### currency

- 当前暂不作为考勤规则核心字段。
- 加班费币种后续由薪资模块统一确认。

## 7. 规则列表设计

### 7.1 展示字段

规则列表展示：

- 规则名称
- 启用状态
- 生效开始时间
- 生效结束时间
- 上班时间
- 下班时间
- 休息时间
- 标准工时
- 是否启用加班
- 加班费标准
- 加班取整说明
- 关联员工数量

### 7.2 操作按钮

每条规则支持：

- 编辑
- 启用/停用
- 查看关联员工
- 查看详情，可与编辑弹窗复用只读模式，后续再做

### 7.3 状态展示

启用状态：

- `isActive = true`：显示“启用”
- `isActive = false`：显示“停用”

有效期状态建议前端派生：

- 未开始：当前日期早于 `effectiveStartDate`
- 生效中：当前日期位于开始和结束之间，或结束为空
- 已过期：当前日期晚于 `effectiveEndDate`

注意：启用状态和有效期状态是两个概念，不要混为一个字段。

## 8. 新增/编辑规则弹窗

### 8.1 表单字段

基础信息：

- 规则名称
- 是否启用

生效时间：

- 生效开始时间
- 生效结束时间，可为空

班次时间：

- 上班时间
- 下班时间
- 休息开始时间
- 休息结束时间
- 标准工时

加班规则：

- 是否启用加班
- 加班费标准
- 加班计量说明，只读展示：按 0.5 小时为最小单位，向下取整

### 8.2 默认值

新增规则默认值建议：

```ts
const DEFAULT_ATTENDANCE_RULE_FORM: AttendanceRuleFormData = {
  name: '',
  isActive: true,
  effectiveStartDate: '',
  effectiveEndDate: null,
  startShift: '08:30',
  endShift: '17:30',
  breakStart: '12:00',
  breakEnd: '13:00',
  standardHours: 8,
  overtimeEnabled: true,
  otHourlyFee: 50,
};
```

### 8.3 前端校验

提交前校验：

- 规则名称必填。
- 生效开始时间必填。
- 生效结束时间可为空。
- 如果填写生效结束时间，不能早于生效开始时间。
- 上班时间必填。
- 下班时间必填。
- 休息开始时间必填。
- 休息结束时间必填。
- 标准工时必须大于 0。
- 启用加班时，加班费标准必须大于等于 0。

### 8.4 编辑已关联规则提示

编辑时，如果接口返回 `relatedEmployeeCount > 0`，保存前展示二次确认。

确认文案：

```text
该考勤规则已被员工引用。保存修改后，关联员工后续考勤计算将按新规则执行。如重新计算历史考勤，历史结果也可能变化。是否继续保存？
```

用户确认后再提交保存请求。

## 9. 查看关联员工弹窗

### 9.1 打开方式

入口：

- 规则列表操作按钮：查看关联员工
- 规则详情操作按钮：查看关联员工

打开时按规则 ID 请求关联员工列表。

### 9.2 展示字段

弹窗展示：

- 员工编号
- 员工姓名
- 部门
- 岗位
- 员工状态
- 关联开始日期
- 关联结束日期
- 当前是否有效关联

### 9.3 空状态

如果没有员工关联：

```text
当前没有员工关联此考勤规则。
```

### 9.4 操作限制

弹窗只负责查看。

不提供：

- 修改员工规则
- 移除员工关联
- 新增员工关联
- 修改员工状态

## 10. 启用/停用交互

### 10.1 启用

点击启用后：

- 调用启用接口。
- 成功后刷新列表。
- 启用不自动检查是否有员工关联。

### 10.2 停用

点击停用前展示确认弹窗。

确认文案：

```text
停用后，该规则不再允许新员工选择或员工新切换到该规则，但已关联员工可继续使用，历史考勤不受影响。是否确认停用？
```

确认后：

- 调用停用接口。
- 成功后刷新列表。
- 不要求员工强制更换规则。

---

## 11. 员工管理联动

员工管理模块选择考勤规则时：

- 只展示 `isActive = true` 的规则。
- 默认不展示停用规则。
- 是否过滤有效期由后端接口统一决定，前端不自行绕过。
- 员工历史关联中仍可展示停用规则名称。

考勤规则模块本身不负责修改员工绑定关系。

---

## 12. 考勤计算联动

前端展示考勤计算结果时，需要知道结果来自哪条规则。

后续考勤结果详情中可展示：

- 使用的考勤规则名称
- 规则 ID
- 上班/下班时间
- 标准工时
- 加班费标准

本模块只维护规则，不直接计算考勤结果。

## 14. 验收标准

前端完成后需要满足：

- 可以查看考勤规则列表。
- 可以新增考勤规则。
- 可以编辑考勤规则。
- 已被员工引用的规则允许编辑，但保存前提示影响。
- 可以启用/停用考勤规则。
- 停用规则不强制员工更换规则。
- 可以通过弹窗查看关联员工。
- 生效结束时间可以为空。
- 不展示迟到、早退、缺勤配置。
- 不在本模块处理税率、币种、员工薪资配置。
