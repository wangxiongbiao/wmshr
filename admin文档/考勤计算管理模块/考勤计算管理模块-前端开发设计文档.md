# 考勤计算管理模块前端开发设计文档

## 1. 文档目标

本文档用于指导 `wmshr-admin` 考勤计算管理模块的前端开发改造。

设计原则：

- 以现有 `AttendanceTable.tsx`、`AttendanceAdjustmentModal`、`calcAttendanceDetails` 已有能力为基础。
- 字段优先使用现有代码字段，缺失字段按业务文档和字段梳理文档补齐。
- 前端负责展示考勤计算结果、触发重算、展示异常、发起人工调整。
- 前端不维护员工档案、不配置考勤规则、不计算最终工资。

关联文档：

- `admin文档/考勤计算管理模块/考勤计算管理模块-字段与功能梳理.md`
- `admin文档/考勤计算管理模块/考勤计算管理模块-后端开发设计文档.md`
- `admin文档/考勤规则管理模块/考勤规则管理模块-前端开发设计文档.md`
- `admin文档/员工原理模块/员工管理模块-前端开发设计文档.md`

## 3. 前端目标数据结构

## 3.1 AttendanceRecord

前端继续兼容现有字段，并补充来源字段。

```ts
export type AttendanceSource = 'manual' | 'device' | 'import' | 'system';

export type AttendanceType =
  | 'normal'
  | 'late'
  | 'early'
  | 'absent'
  | 'leave'
  | 'overtime';

export interface AttendanceRecord {
  id: string | number;
  employeeId: string | number;
  date: string;
  inTime: string | null;
  outTime: string | null;
  type: AttendanceType;
  note: string;
  source: AttendanceSource;
}
```

兼容说明：

- 当前代码使用 `empId`，后续接口建议统一为 `employeeId`。
- 前端适配阶段可以在接口层做字段映射，避免一次性大范围改组件。

## 4. 页面结构设计

考勤计算页面建议拆成三个区域：

1. 筛选和操作区
2. 每日考勤计算结果列表
3. 月度汇总区

页面内弹窗：

- 每日结果详情弹窗
- 调整考勤记录弹窗
- 重新计算确认弹窗
- 异常说明弹窗，可与详情弹窗合并

---

## 5. 筛选和操作区

### 5.1 筛选字段

建议支持：

- 年月：`YYYY-MM`
- 员工
- 考勤状态
- 是否异常

### 5.2 操作按钮

建议支持：

- 查询
- 重置筛选
- 重新计算当前月份
- 重新计算选中记录

### 5.3 默认筛选

默认值：

- 年月：当前月份
- 员工：全部
- 状态：全部
- 是否异常：全部

## 6. 每日考勤计算列表

### 6.1 展示字段

建议表格字段：

- 日期
- 员工编号
- 员工姓名
- 使用考勤规则
- 上班时间
- 下班时间
- 原始工时
- 休息扣除
- 有效工时
- 标准工时
- 加班原始时长
- 加班计薪时长
- 状态
- 异常原因
- 操作

### 6.2 字段展示规则

#### 使用考勤规则

- 有规则时展示规则名称。
- 没有规则时展示 `未匹配规则`，并标记异常。

#### 原始工时

- 展示 `rawHours`。
- 缺勤、假期或异常无打卡时展示 `0.00h` 或 `-`，以详情为准。

#### 休息扣除

- 展示 `breakDeductionHours`。
- 用于解释有效工时如何得出。

#### 加班计薪时长

- 展示 `overtimePayHours`。
- 注意该字段必须是后端按 0.5 小时向下取整后的结果。
- 前端不自行重新计算，以避免口径不一致。

#### 异常原因

- `hasException = true` 时展示红色标记。
- `exceptionReason` 为空时展示 `异常原因未返回`。

### 6.3 操作按钮

每行支持：

- 查看详情
- 调整记录
- 重新计算该日

## 8. 调整考勤记录弹窗

当前已有 `AttendanceAdjustmentModal`，后续继续复用并补充。

### 8.1 表单字段

保留现有字段：

- 日期
- 考勤类型
- 上班时间
- 下班时间
- 调整备注

补充字段：

- 考勤来源，保存人工调整时固定为 `manual`

### 8.2 保存行为

保存后流程：

1. 提交原始考勤记录调整。
2. 后端保存调整记录。
3. 后端触发该员工该日期重新计算，或前端再调用单日重算接口。
4. 前端刷新每日计算结果列表。
5. 前端刷新月度汇总。

### 8.3 限制

- 调整记录不修改员工档案。
- 调整记录不修改考勤规则。
- 调整记录不直接修改薪资结果。

`canGeneratePayroll = true`：

```text

```

`canGeneratePayroll = false`：

```text
不可进入薪资核算：{blockedReason}
```

### 9.3 与薪资核算联动

- 前端薪资核算页面后续应读取月度汇总，而不是直接遍历原始考勤记录。
- 考勤计算页面只展示汇总状态，不生成工资。

## 10. 重新计算交互

### 10.1 单日重算

入口：每日结果行操作。

确认文案：

```text
将重新计算该员工该日期的考勤结果，已有计算结果会被刷新。是否继续？
```

### 10.2 选中记录重算

入口：批量操作。

确认文案：

```text
将重新计算选中的考勤结果，已有计算结果会被刷新。是否继续？
```

### 10.3 月度重算

入口：页面顶部按钮。

确认文案：

```text
将重新计算当前筛选月份的考勤结果。若考勤规则已被修改，历史结果可能发生变化。是否继续？
```

### 10.4 重算后刷新

重算成功后刷新：

- 每日结果列表
- 月度汇总
- 异常数量

## 11. 异常展示

### 11.1 异常类型

前端需要支持展示：

- 找不到员工
- 找不到有效考勤规则
- 上班时间缺失
- 下班时间缺失
- 时间格式错误
- 规则已过期

### 11.2 异常样式

建议：

- 行背景浅红
- 状态标签显示“异常”
- 异常原因列展示具体原因
- 月度汇总中累计异常数量

### 11.3 异常处理入口

异常行可操作：

- 查看详情
- 调整记录
- 重新计算

## 13. 前端改造建议

### 13.1 保留现有组件能力

保留：

- `AttendanceTable` 的表格结构
- `AttendanceAdjustmentModal` 的基本表单
- `formatDuration`
- 状态标签展示

### 13.2 改造方向

建议后续改造：

- `AttendanceTable` 数据源从 `AttendanceRecord[]` 改为 `AttendanceCalculationResult[]`。
- 表格不再直接调用 `calcAttendanceDetails`。
- `calcAttendanceDetails` 可保留为前端临时预览或本地演示逻辑，但正式结果以后端返回为准。
- 增加月度汇总组件。
- 增加详情弹窗和重算确认弹窗。

---

## 14. 验收标准

前端完成后需要满足：

- 可以按年月查看每日考勤计算结果。
- 可以看到使用的考勤规则。
- 可以看到原始工时、休息扣除、有效工时、标准工时。
- 可以看到加班原始时长和加班计薪时长。
- 加班计薪时长展示后端 0.5 小时向下取整结果。
- 可以查看每日结果详情。
- 可以调整原始考勤记录。
- 调整后可以刷新或触发重算。
- 可以重新计算单日、选中记录、整月考勤。
- 可以查看月度汇总。
- 可以看到异常原因。
- 异常结果会阻断进入薪资核算。
- 不在本模块编辑员工档案、考勤规则或薪资结果。
